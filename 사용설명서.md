## 설치

### 1. uv 설치 (패키지 매니저)

```powershell
pip install uv
```

### 2. 저장소 클론 및 의존성 설치

```powershell
git clone https://github.com/GonGe1018/gentuber-v1
cd gentuber-v1
uv sync
```

`uv sync` 한 번으로 PyTorch (CUDA 12.8), MediaPipe, OpenCV 등 모든 의존성이 자동 설치됩니다.

### 3. 모델 자동 다운로드

첫 실행 시 필요한 모델이 `~/.cache/huggingface/`에 자동으로 다운로드됩니다 (총 약 4 GB).

## 빠른 시작

### 웹캠으로 실행

```powershell
uv run gentuber
```

기본값으로 웹캠(인덱스 0)을 입력으로 사용하고, 256×256 해상도, IP-Adapter + ControlNet + LCM 백엔드로 실행됩니다. 설정 GUI가 먼저 표시됩니다.

### GUI 없이 실행

```powershell
uv run gentuber --no-gui --source 0
```

### 영상 파일 → MP4 저장

```powershell
uv run gentuber --no-gui --source input.mp4 --output result.mp4
```

### 종료

실행 중 표시 창에서 `q` 키를 누르면 종료됩니다.

## 실행 방법

### 품질 프리셋

```powershell
# 빠른 모드: 256px, 손 감지 없음
uv run gentuber --quality fast

# 균형 모드: 384px
uv run gentuber --quality balanced

# 고품질 모드: 512px
uv run gentuber --quality quality
```

### 해상도 직접 지정

```powershell
uv run gentuber --size 256    # 빠름 (기본값)
uv run gentuber --size 384    # 균형
uv run gentuber --size 512    # 고품질
```

### 캐릭터 레퍼런스 이미지 변경

```powershell
uv run gentuber --reference my_character.png
```

### IP-Adapter / ControlNet 강도 조절

```powershell
# 강한 캐릭터 보존, 적당한 포즈
uv run gentuber --ip-scale 0.6 --cn-scale 1.0

# 강한 포즈 반영, 약한 캐릭터
uv run gentuber --ip-scale 0.4 --cn-scale 2.0
```

### Temporal Feedback 조절

```powershell
# 강한 일관성 (기본값)
uv run gentuber --feedback 0.2

# 피드백 없이 (각 프레임 독립)
uv run gentuber --feedback 1.0
```

### 프롬프트 변경

```powershell
uv run gentuber --prompt "anime girl, blue hair, school uniform, white background"
uv run gentuber --negative-prompt "realistic, 3d, photo, blurry"
```

## CLI 옵션 전체 목록

```powershell
uv run gentuber [옵션]
```

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--source` | `0` | 입력 소스 (웹캠 인덱스 또는 영상 파일 경로) |
| `--output`, `-o` | — | MP4로 저장 후 종료 (헤드리스, GUI 없음) |
| `--reference` | `assets/reference.png` | IP-Adapter용 캐릭터 레퍼런스 이미지 |
| `--size` | `256` | 출력 해상도: `256` / `384` / `512` |
| `--quality` | — | 프리셋: `fast` / `balanced` / `quality` |
| `--steps` | `4` | 추론 스텝 수 |
| `--ip-scale` | `0.6` | IP-Adapter 강도 (0.3=약함, 0.7=강한 캐릭터 보존) |
| `--cn-scale` | `1.8` | ControlNet 포즈 강도 |
| `--guidance` | `1.2` | Guidance scale |
| `--feedback` | `0.2` | Temporal feedback (0.2=강한 일관성, 1.0=피드백 없음) |
| `--prompt` | config.py 참고 | 생성 프롬프트 |
| `--negative-prompt` | config.py 참고 | 네거티브 프롬프트 |
| `--seed` | `42` | 노이즈 시드 (`-1`이면 매 실행마다 랜덤) |
| `--half-body` | 꺼짐 | VTuber 모드: 상체만 표시 |
| `--max-fps` | `60` | 화면 표시 최대 FPS 제한 (`0`이면 무제한) |
| `--no-gui` | 꺼짐 | 설정 GUI 건너뛰기 |
| `--no-skeleton` | 꺼짐 | 스켈레톤 오버레이 숨기기 |
| `--no-interp` | 켜짐 | 프레임 간 시간적 스무딩 비활성화 |
| `--no-hands` | 꺼짐 | 손 랜드마크 감지 건너뜀 |
| `--motion-lo` | `0.008` | 적응형 모션: 떨림 임계값 |
| `--motion-hi` | `0.04` | 적응형 모션: 큰 움직임 리셋 임계값 |
| `--motion-max` | `0.85` | 적응형 모션: 최대 피드백 강도 상한 |

### 사용 예시

```powershell
# 웹캠, 고품질
uv run gentuber --size 512 --no-hands

# 영상 파일 → MP4 저장
uv run gentuber --no-gui --source dance.mp4 -o dance_anime.mp4 --steps 4

# 강한 캐릭터 보존
uv run gentuber --ip-scale 0.7 --cn-scale 1.0

# VTuber 반샷 모드
uv run gentuber --half-body

# 스무딩 없이 원본 출력
uv run gentuber --no-interp --max-fps 0
```

## 런타임 튜닝

라이브 모드(`--output` 없이 실행)에서는 영상 창 옆에 튜닝 패널이 자동으로 열립니다.

슬라이더로 조절 가능한 파라미터:
- IP-Adapter Scale (캐릭터 강도)
- ControlNet Scale (포즈 강도)
- Guidance Scale
- Feedback Strength (시간적 일관성)
- Motion Lo/Hi (적응형 모션 임계값)
- Motion Max Strength

슬라이더 조절 후 Apply 클릭하면 모델 재로딩 없이 즉시 반영됩니다.

## config.py 설정

CLI 옵션 대신 `config.py`를 직접 수정해 기본값을 바꿀 수 있습니다.

```python
# config.py

cfg.video_source = "0"                      # 기본 입력을 웹캠으로
cfg.reference_image = "my_character.png"    # 캐릭터 이미지
cfg.ip_adapter_scale = 0.6                  # 캐릭터 강도
cfg.controlnet_conditioning_scale = 1.8     # 포즈 강도
cfg.guidance_scale = 1.2                    # Guidance scale
cfg.temporal_feedback_strength = 0.2        # 스타일 일관성
cfg.num_inference_steps = 4                 # 추론 스텝 수

cfg.output_width = 256                      # 기본 해상도
cfg.output_height = 256
cfg.capture_width = 256
cfg.capture_height = 256

cfg.seed = 42                               # 노이즈 시드 (-1 = 랜덤)
cfg.detect_hands = True                     # 손 감지
cfg.half_body = False                       # VTuber 반샷 모드
cfg.no_interp = True                        # 프레임 스무딩 비활성화

cfg.prompt = "anime girl, blue hair, school uniform, white background"
cfg.negative_prompt = "realistic, 3d, photo, blurry, low quality"
```

### 프롬프트 작성 팁

```python
# 좋은 프롬프트 예시
cfg.prompt = (
    "masterpiece, best quality, anime girl, full body, "
    "colorful outfit, white background, 2d illustration, "
    "sharp focus, vibrant colors"
)

# 네거티브 프롬프트 (원하지 않는 요소 제거)
cfg.negative_prompt = (
    "lowres, bad anatomy, bad hands, missing fingers, "
    "blurry, low quality, realistic, 3d, photo, watermark"
)
```

## 모델 다운로드 안내

첫 실행 시 자동으로 다운로드됩니다. 인터넷 연결이 필요합니다.

| 모델 | 용량 | 용도 |
|---|---|---|
| `KBlueLeaf/kohaku-v2.1` | ~2 GB | 애니메이션 SD1.5 베이스 모델 |
| `latent-consistency/lcm-lora-sdv1-5` | ~200 MB | LCM-LoRA 고속 추론 |
| `h94/IP-Adapter` (ip-adapter-plus_sd15) | ~98 MB | 캐릭터 외형 보존 |
| `h94/IP-Adapter` (image_encoder) | ~1.2 GB | CLIP ViT-H-14 (캐싱 후 언로드) |
| `lllyasviel/control_v11p_sd15_openpose` | ~361 MB | ControlNet OpenPose |
| `madebyollin/taesd` | ~5 MB | 경량 VAE 디코더 |
| MediaPipe 포즈/손 모델 | ~14 MB | 포즈 추출 |

다운로드 경로: `C:\Users\사용자명\.cache\huggingface\`

## 자주 묻는 질문

**Q. 첫 실행이 너무 오래 걸려요.**

A. 모델 다운로드(최초 1회, 약 4 GB)와 파이프라인 워밍업(매 실행, 약 10~20초) 때문입니다. 두 번째 실행부터는 다운로드 없이 워밍업만 기다리면 됩니다.

---

**Q. CUDA out of memory 오류가 나요.**

A. 해상도를 낮추거나 손 감지를 끄세요.

```powershell
uv run gentuber --size 256 --no-hands
```

---

**Q. 화면이 너무 흔들려요 / 깜빡여요.**

A. Temporal feedback 강도를 낮추세요 (값이 작을수록 이전 프레임을 더 많이 유지):

```powershell
uv run gentuber --feedback 0.15
```

---

**Q. 스켈레톤이 화면에 보여서 거슬려요.**

A.
```powershell
uv run gentuber --no-skeleton
```

---

**Q. 웹캠 인덱스를 모르겠어요.**

A. 0부터 시작해서 하나씩 올려보세요. 대부분 내장 웹캠은 0, 외장 웹캠은 1입니다.

```powershell
uv run gentuber --source 0
uv run gentuber --source 1
```

---

**Q. 생성 이미지가 매번 달라요. 고정하고 싶어요.**

A. 시드를 고정하세요.

```powershell
uv run gentuber --seed 42
```

---

**Q. 캐릭터가 원본과 너무 달라요.**

A. IP-Adapter 강도를 높이세요.

```powershell
uv run gentuber --ip-scale 0.7
```

---

**Q. 포즈를 더 정확하게 따라가게 하고 싶어요.**

A. ControlNet 강도를 높이세요.

```powershell
uv run gentuber --cn-scale 2.0
```

## 파이프라인 구조

```
웹캠/영상 파일
    ↓
[VideoCapture 스레드]  — 프레임을 링 버퍼에 지속 저장
    ↓
[Pose 스레드]          — MediaPipe로 포즈 추출 → CHW float16 변환
    ↓  pose_queue
[DiffusionEngine 스레드] — IP-Adapter + ControlNet + LCM으로 이미지 생성 (GPU)
    ↓  out_queue
[메인 스레드]          — 프레임 스무딩 + OpenCV 화면 표시
```

각 단계는 독립 스레드에서 실행되며, 큐가 가득 차면 오래된 프레임을 버려 레이턴시가 쌓이지 않습니다.
