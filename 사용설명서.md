## 시스템 요구사항

| 항목 | 최소 | 권장 |
|---|---|---|
| OS | Windows 10/11 | Windows 11 |
| Python | 3.12 | 3.12 |
| CUDA | 12.8 | 12.8 |
| GPU | RTX 3080 (10GB) | RTX 4090 / RTX 5070 Ti |
| RAM | 16 GB | 32 GB |
| 저장공간 | 10 GB (모델 포함) | 20 GB |

> **참고:** `torch.compile`은 Triton이 필요해 Windows에서 동작하지 않습니다. Linux에서는 추가 성능 향상이 가능합니다.

## 설치

### 1. uv 설치 (패키지 매니저)

```powershell
pip install uv
```

### 2. 저장소 클론 및 의존성 설치

```powershell
git clone https://github.com/yourname/realtime-live2d
cd realtime-live2d
uv sync
```

`uv sync` 한 번으로 PyTorch (CUDA 12.8), MediaPipe, OpenCV 등 모든 의존성이 자동 설치됩니다. 별도의 pip 명령이 필요 없습니다.

### 3. 모델 자동 다운로드

첫 실행 시 필요한 모델이 `~/.cache/huggingface/`에 자동으로 다운로드됩니다 (총 약 6~7 GB).

## 빠른 시작

### 테스트 영상으로 바로 실행

```powershell
uv run live2d
```

기본값으로 `assets/test_input.mp4`를 입력으로 사용하고, 384×384 해상도, KohakuV2 + LCM-LoRA 백엔드로 실행됩니다.

### 웹캠으로 실행

```powershell
uv run live2d --source 0
```

`0`은 기본 웹캠 인덱스입니다. 웹캠이 여러 개라면 `1`, `2` 등으로 변경하세요.

### 종료

실행 중 표시 창에서 `q` 키를 누르면 종료됩니다.

## 실행 방법

### 품질 프리셋 (가장 간단한 방법)

속도와 품질의 균형을 빠르게 조절할 수 있습니다.

```powershell
# 빠른 모드: 256px, 손 감지 없음, ~132 FPS
uv run live2d --quality fast

# 균형 모드: 384px, 기본값, ~73 FPS
uv run live2d --quality balanced

# 고품질 모드: 512px, ~47 FPS
uv run live2d --quality quality
```

### 해상도 직접 지정

```powershell
uv run live2d --size 256    # 빠름
uv run live2d --size 384    # 기본값
uv run live2d --size 512    # 고품질
```

### 입력 소스 변경

```powershell
# 웹캠 (인덱스 0)
uv run live2d --source 0

# 영상 파일
uv run live2d --source "C:\Videos\dance.mp4"
```

### 백엔드 변경

```powershell
# KohakuV2 + LCM-LoRA (기본값, 애니메이션 특화, ~73 FPS)
uv run live2d --backend lcm_graph

# SD-Turbo (범용, ~77 FPS)
uv run live2d --backend sdturbo_graph
```

### 프롬프트 변경

```powershell
uv run live2d --prompt "anime girl, blue hair, school uniform, white background"
uv run live2d --negative-prompt "realistic, 3d, photo, blurry"
```

### 편의 스크립트 사용

```powershell
.\run.ps1 --source 0 --size 384
.\run.ps1 --quality fast
```

## CLI 옵션 전체 목록

```powershell
uv run live2d [옵션]
```

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--source` | `assets/test_input.mp4` | 입력 소스 (웹캠 인덱스 또는 영상 파일 경로) |
| `--backend` | `lcm_graph` | 추론 백엔드 선택 (아래 백엔드 가이드 참고) |
| `--size` | `384` | 출력 해상도: `256` / `384` / `512` |
| `--quality` | — | 프리셋: `fast` / `balanced` / `quality` |
| `--model` | — | `lcm_graph` 백엔드용 애니메이션 모델 (HuggingFace ID) |
| `--steps` | `1` | LCM 추론 스텝 수 (1~4, `t2i`/`controlnet` 백엔드에만 적용) |
| `--prompt` | config.py 참고 | 생성 프롬프트 |
| `--negative-prompt` | config.py 참고 | 네거티브 프롬프트 |
| `--seed` | `42` | 노이즈 시드 (`-1`이면 매 실행마다 랜덤) |
| `--max-fps` | `60` | 화면 표시 최대 FPS 제한 (`0`이면 무제한) |
| `--no-skeleton` | 꺼짐 | 스켈레톤 오버레이 숨기기 |
| `--no-interp` | 꺼짐 | 프레임 간 시간적 스무딩 비활성화 |
| `--no-hands` | 꺼짐 | 손 랜드마크 감지 건너뜀 (프레임당 ~6ms 절약) |

### 사용 예시

```powershell
# 웹캠, 512px 고품질, 손 감지 없음
uv run live2d --source 0 --size 512 --no-hands

# 영상 파일, 커스텀 프롬프트, 시드 고정
uv run live2d --source dance.mp4 --prompt "anime girl, red dress" --seed 123

# SD-Turbo 백엔드, 256px, 스켈레톤 숨김
uv run live2d --backend sdturbo_graph --size 256 --no-skeleton

# DreamShaper 모델로 변경
uv run live2d --model "Lykon/dreamshaper-8"

# 스무딩 없이 원본 출력, FPS 무제한
uv run live2d --no-interp --max-fps 0
```

## 백엔드 선택 가이드

### RTX 5070 Ti 기준 성능표

| 백엔드 | 256×256 | 384×384 | 512×512 | 특징 |
|---|---|---|---|---|
| `lcm_graph` | ~103 FPS | ~60 FPS | ~37 FPS | 애니메이션 특화, 최고 품질 ★★★★ |
| `sdturbo_graph` | ~104 FPS | ~63 FPS | ~40 FPS | 범용, 약간 빠름 ★★★ |
| `sdturbo` | ~28 FPS | ~26 FPS | — | 이거 쓸 이유 없음 (graph 버전 사용) |
| `t2i` | — | ~27 FPS | ~15 FPS | 멀티스텝 지원 |
| `controlnet` | — | ~20 FPS | ~11 FPS | 가장 느림 |

### 추천

- **일반 사용:** `lcm_graph` (기본값) — 애니메이션 스타일에 최적화
- **더 빠른 속도:** `sdturbo_graph` — 384px 기준 약 5% 빠름
- **다른 모델 시도:** `--model "Lykon/dreamshaper-8"` 등 SD1.5 호환 모델 사용 가능

### 백엔드별 특이사항

**`lcm_graph` / `sdturbo_graph`**
- CUDA 그래프로 캡처된 고속 경로 사용
- `--steps 1` 고정 (CUDA 그래프는 루프를 캡처할 수 없음)
- 첫 실행 시 그래프 캡처로 약 10~20초 워밍업 필요

**`t2i` / `controlnet`**
- `--steps 1~4` 조절 가능
- 스텝 수가 많을수록 품질 향상, 속도 감소
- `--steps 2`가 품질/속도 균형에 좋음

## config.py 설정

CLI 옵션 대신 `config.py`를 직접 수정해 기본값을 바꿀 수 있습니다.

```python
# config.py

cfg.video_source = 0                    # 기본 입력을 웹캠으로 변경
cfg.engine_backend = "lcm_graph"        # 기본 백엔드
cfg.lcm_model_id = "Lykon/dreamshaper-8"  # 기본 애니메이션 모델 변경

cfg.output_width = 512                  # 기본 해상도 512px로 변경
cfg.output_height = 512
cfg.capture_width = 512
cfg.capture_height = 512

cfg.num_inference_steps = 2             # t2i/controlnet 백엔드 스텝 수
cfg.seed = -1                           # 매 실행마다 랜덤 시드

cfg.detect_hands = False                # 손 감지 비활성화 (속도 향상)
cfg.interp_alpha = 0.5                  # 스무딩 강도 (0.0~1.0)

cfg.prompt = "anime girl, blue hair, school uniform, white background"
cfg.negative_prompt = "realistic, 3d, photo, blurry, low quality"

cfg.show_skeleton_overlay = False       # 스켈레톤 오버레이 숨기기
cfg.show_fps = True                     # FPS 표시
cfg.max_fps = 0                         # FPS 제한 없음
```

### 프롬프트 작성 팁

```python
# 좋은 프롬프트 예시
cfg.prompt = (
    "masterpiece, best quality, anime girl, full body, "
    "colorful outfit, white background, 2d illustration, "
    "sharp focus, vibrant colors"
)

# 네거티브 프롬프트 (원하지 않는 요소 제거)
cfg.negative_prompt = (
    "lowres, bad anatomy, bad hands, missing fingers, "
    "blurry, low quality, realistic, 3d, photo, watermark"
)
```

## 모델 다운로드 안내

첫 실행 시 자동으로 다운로드됩니다. 인터넷 연결이 필요합니다.

| 모델 | 용량 | 용도 |
|---|---|---|
| `KBlueLeaf/kohaku-v2.1` | ~2 GB | 애니메이션 SD1.5 베이스 (`lcm_graph` 기본값) |
| `latent-consistency/lcm-lora-sdv1-5` | ~200 MB | LCM-LoRA 가중치 |
| `stabilityai/sd-turbo` | ~3.1 GB | SD-Turbo 베이스 (`sdturbo_graph`) |
| `TencentARC/t2iadapter_openpose_sd14v1` | ~77 MB | 포즈 컨디셔닝 |
| `madebyollin/taesd` | ~5 MB | 경량 VAE 디코더 |
| `pose_landmarker_lite.task` | ~5 MB | MediaPipe 포즈 모델 |
| `hand_landmarker.task` | ~9 MB | MediaPipe 손 모델 |

다운로드 경로: `C:\Users\사용자명\.cache\huggingface\`

### 다른 애니메이션 모델 사용하기

HuggingFace에 있는 SD1.5 호환 모델이면 `lcm_graph` 백엔드에서 사용 가능합니다.

```powershell
# DreamShaper 8
uv run live2d --model "Lykon/dreamshaper-8"

# Anything V5
uv run live2d --model "stablediffusionapi/anything-v5"

# Counterfeit V3
uv run live2d --model "gsdf/Counterfeit-V3.0"
```

## 자주 묻는 질문

**Q. 첫 실행이 너무 오래 걸려요.**

A. 모델 다운로드(최초 1회, 약 6~7 GB)와 CUDA 그래프 캡처(매 실행, 약 10~20초) 때문입니다. 두 번째 실행부터는 다운로드 없이 그래프 캡처만 기다리면 됩니다.

---

**Q. CUDA out of memory 오류가 나요.**

A. 해상도를 낮추거나 손 감지를 끄세요.

```powershell
uv run live2d --size 256 --no-hands
```

---

**Q. 화면이 너무 흔들려요 / 깜빡여요.**

A. 시간적 스무딩 강도를 높이세요. `config.py`에서:

```python
cfg.interp_alpha = 0.2  # 낮을수록 더 부드럽지만 반응이 느려짐
```

또는 CLI에서:
```powershell
# 스무딩 완전히 끄기 (원본 출력)
uv run live2d --no-interp
```

---

**Q. 스켈레톤이 화면에 보여서 거슬려요.**

A.
```powershell
uv run live2d --no-skeleton
```

---

**Q. 웹캠 인덱스를 모르겠어요.**

A. 0부터 시작해서 하나씩 올려보세요. 대부분 내장 웹캠은 0, 외장 웹캠은 1입니다.

```powershell
uv run live2d --source 0
uv run live2d --source 1
```

---

**Q. 생성 이미지가 매번 달라요. 고정하고 싶어요.**

A. 시드를 고정하세요.

```powershell
uv run live2d --seed 42
```

---

**Q. 더 애니메이션스러운 스타일을 원해요.**

A. 프롬프트를 조정하거나 다른 모델을 사용해보세요.

```powershell
uv run live2d --model "Lykon/dreamshaper-8" --prompt "anime style, vibrant colors, detailed"
```

---

**Q. `torch.compile` 관련 오류가 나요.**

A. Windows에서는 Triton이 없어 `torch.compile`이 동작하지 않습니다. 이미 CUDA 그래프로 최적화되어 있으므로 별도 조치가 필요 없습니다.

---

**Q. INT8 양자화를 써보고 싶어요.**

A. Windows에서 bitsandbytes가 정상 동작하지 않아 오히려 느려집니다. 현재 float16 + CUDA 그래프 조합이 최적입니다.

## 파이프라인 구조

```
웹캠/영상 파일
    ↓
[VideoCapture 스레드]  — 프레임을 링 버퍼에 지속 저장
    ↓
[Pose 스레드]          — MediaPipe로 포즈 추출 → CHW float16 변환
    ↓  pose_queue
[DiffusionEngine 스레드] — CUDA 그래프로 이미지 생성 (GPU)
    ↓  out_queue
[메인 스레드]          — 프레임 스무딩 + OpenCV 화면 표시
```

각 단계는 독립 스레드에서 실행되며, 큐가 가득 차면 오래된 프레임을 버려 레이턴시가 쌓이지 않습니다.
